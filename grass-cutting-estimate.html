⚙️ データ管理

☁️ GitHub バックアップ データを失わない安心設定

ブラウザのlocalStorageは消える可能性があります。GitHubに保存することで、いつでも復元可能になります。
初回のみ Personal Access Token (PAT) の登録が必要です。

最終保存: 未設定
📖 GitHub Token の作成方法（クリックで展開）
  1. https://github.com/settings/tokens?type=beta を開く
  2. 「Generate new token」をクリック
  3. Token name: kusakari-data など
  4. Expiration: No expiration または好きな期間
  5. Repository access: Only select repositories → kusakari-site を選択
  6. Permissions → Repository permissions:
    Contents: Read and write に設定
  7. 「Generate token」→ 出てきた github_pat_xxxxx をコピーして上の欄に貼り付け

💴 機械別 時給

⚡ 機械別 作業効率 (㎡/h)

見積時の「担当㎡⇔時間」変換に使用。地形難易度が高い時は自動的に時間が増えます。(例 平地×1.0 / 傾斜×3.0)

📐 面積ベース見積 基準単価

面積ベース見積の算出基準。面積 × この単価 = 基準作業費。機械ベース(時間×時給)と照合して妥当性をチェック。

🤝 委託設定

委託先には「合計額×%」を支払い、残りが自分の粗利。機械ごとの配分も試算表に表示されます。

🛡 最低料金

どんな小さな案件でも最低これだけは請求する金額(全案件に適用)

📅 作業スケジュール 同エリア優先・移動最小

「作業予定日」を入れた案件が日付ごとに自動整理されます。エリア順にソートされ、移動が最小になるよう一覧表示します。

👥 メンバー管理

メンバーごとに使用可能な機械（スキル）と稼働状況を登録。スケジュール時の人員割当に使用されます。

🛠 機械×メンバー対応マトリクス

👥 担当者・機械・面積を割当

担当者ごとに「機械」と「担当面積」を割当てます。
時間と委託金は自動計算されます（時間 = 面積 × 補正 ÷ 機械効率、金額 = 時間 × 時給）。
